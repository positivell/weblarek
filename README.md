# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

# Модели и интерфейсы

## Интерфейсы

### `ApiPostMethods`
Тип, представляющий методы HTTP-запросов для отправки данных:
- `'POST'`
- `'PUT'`
- `'DELETE'`

### `IApi`
Интерфейс для работы с API.

Методы:
- `get<T>(uri: string): Promise<T>` — выполняет GET-запрос на указанный эндпоинт и возвращает данные.
- `post<T>(uri: string, data: object, method?: ApiPostMethods): Promise<T>` — выполняет запрос на указанный эндпоинт с переданными данными. Метод запроса по умолчанию — `POST`.

---

### `TPayment`
Тип, представляющий способы оплаты:
- `'card'` — оплата картой.
- `'cash'` — оплата наличными.
- `""` — неопределённый способ оплаты.

---

### `IProduct`
Интерфейс, описывающий товар.

Поля:
- `id: string` — уникальный идентификатор товара.
- `description: string` — описание товара.
- `image: string` — ссылка на изображение товара.
- `title: string` — название товара.
- `category: string` — категория товара.
- `price: number | null` — цена товара.

---

### `IBuyer`
Интерфейс, описывающий покупателя.

Поля:
- `payment: TPayment` — способ оплаты.
- `email: string` — email покупателя.
- `phone: string` — телефон покупателя.
- `address: string` — адрес доставки.

---

### `IOrder`
Интерфейс, описывающий заказ.

Поля:
- `payment: TPayment` — способ оплаты.
- `email: string` — email покупателя.
- `phone: string` — телефон покупателя.
- `address: string` — адрес доставки.
- `total: number` — общая сумма заказа.
- `items: string[]` — массив идентификаторов товаров.

---

### `IOrderResult`
Интерфейс, описывающий результат оформления заказа.

Поля:
- `id: string` — уникальный идентификатор заказа.
- `total: number` — общая сумма заказа.

---

### `IProductsResponse`
Интерфейс, описывающий ответ сервера с товарами.

Поля:
- `total: number` — общее количество товаров.
- `items: IProduct[]` — массив товаров.

---

## Модели

### `ProductsModel`
Модель для работы с товарами.

Поля:
- `items: IProduct[]` — массив товаров.
- `preview: IProduct | null` — текущий товар для предпросмотра.

Методы:
- `setItems(items: IProduct[]): void` — устанавливает массив товаров.
- `getItems(): IProduct[]` — возвращает массив товаров.
- `getProductById(id: string): IProduct | undefined` — возвращает товар по его идентификатору.
- `setPreview(product: IProduct | null | undefined): void` — устанавливает товар для предпросмотра.
- `getPreview(): IProduct | null` — возвращает текущий товар для предпросмотра.

---

### `BuyerModel`
Модель для работы с данными покупателя.

Поля:
- `buyer: IBuyer` — объект с данными покупателя.

Методы:
- `setPayment(payment: TPayment): void` — устанавливает способ оплаты.
- `setAddress(address: string): void` — устанавливает адрес доставки.
- `setEmail(email: string): void` — устанавливает email покупателя.
- `setPhone(phone: string): void` — устанавливает телефон покупателя.
- `getAll(): IBuyer` — возвращает объект с данными покупателя.
- `clear(): void` — очищает данные покупателя.
- `validate(): Partial<Record<keyof IBuyer, string>>` — проверяет корректность данных покупателя и возвращает объект с ошибками.

---

### `BasketModel`
Модель для работы с корзиной.

Поля:
- `_items: IProduct[]` — массив товаров в корзине.

Методы:
- `getItems(): IProduct[]` — возвращает массив товаров в корзине.
- `addItem(product: IProduct): void` — добавляет товар в корзину.
- `removeItem(id: string): void` — удаляет товар из корзины по его идентификатору.
- `clear(): void` — очищает корзину.
- `getTotalPrice(): number` — возвращает общую стоимость товаров в корзине.
- `getCount(): number` — возвращает количество товаров в корзине.
- `contains(id: string): boolean` — проверяет, есть ли товар с указанным идентификатором в корзине.

---

### `LarekApi`
Модель для работы с API интернет-магазина.

Поля:
- `_api: Api` — экземпляр класса `Api`.

Методы:
- `getProducts(): Promise<IProductsResponse>` — выполняет запрос на получение списка товаров.
- `postOrder(order: IOrder): Promise<IOrderResult>` — отправляет заказ на сервер.

## Views (Представления)

### `GalleryView`
Компонент для отображения галереи товаров.

**Конструктор:**
- `constructor(container: HTMLElement)` — принимает контейнер, в котором будет отображаться галерея.

**Методы:**
- `render(items: HTMLElement[]): void` — принимает массив DOM-элементов и заменяет содержимое контейнера на переданные элементы.

---

### `CardCatalog`
Компонент для отображения карточки товара в каталоге.

**Конструктор:**
- `constructor(container: HTMLElement, event: { onClick: (event: MouseEvent) => void })` — принимает контейнер для карточки и объект с обработчиком события клика.

**Сеттеры:**
- `category: string` — устанавливает категорию товара и добавляет соответствующий класс.
- `image: string` — устанавливает URL изображения товара.

---

### `CardPreview`
Компонент для отображения подробной информации о товаре.

**Конструктор:**
- `constructor(container: HTMLElement, events: IEvents)` — принимает контейнер для отображения карточки и объект событий.

**Сеттеры:**
- `image: string` — устанавливает URL изображения товара.
- `price: number | null` — устанавливает цену товара. Если цена `null`, кнопка добавления в корзину становится неактивной.
- `description: string` — устанавливает описание товара.
- `category: string` — устанавливает категорию товара и добавляет соответствующий класс.
- `productInCart: boolean` — изменяет текст кнопки в зависимости от того, находится ли товар в корзине.

---

### `Basket`
Компонент для отображения корзины.

**Конструктор:**
- `constructor(container: HTMLElement, events: IEvents)` — принимает контейнер для корзины и объект событий.

**Сеттеры:**
- `items: HTMLElement[]` — устанавливает список товаров в корзине.
- `total: number` — устанавливает общую стоимость товаров в корзине.

---

### `CardBasket`
Компонент для отображения товара в корзине.

**Конструктор:**
- `constructor(container: HTMLElement, events: EventEmitter, product: IProduct)` — принимает контейнер для карточки, объект событий и данные о товаре.

**Сеттеры:**
- `index: number` — устанавливает индекс товара в списке корзины.

---

### `Order`
Компонент для отображения формы заказа.

**Конструктор:**
- `constructor(container: HTMLElement, actions: { onInput: (name: string, value: string) => void; onPayment: (method: TPayment) => void; onSubmit: () => void })` — принимает контейнер для формы и объект с обработчиками событий.

**Сеттеры:**
- `address: string` — устанавливает адрес доставки.

### `Contact`
Компонент для отображения формы контактных данных.

**Конструктор:**
- `constructor(container: HTMLElement, actions: { onInput: (name: string, value: string) => void; onSubmit: () => void })` — принимает контейнер для формы и объект с обработчиками событий.

**Сеттеры:**

---

## События (Events)

### Общие события:
- `products:changed` — вызывается при изменении списка товаров.
- `product:select` — вызывается при выборе товара (передается `id` товара).
- `product:selected` — вызывается при выборе товара для предпросмотра (передается объект товара).
- `product:cartAction` — вызывается при добавлении/удалении товара из корзины.
- `cart:changed` — вызывается при изменении содержимого корзины.
- `basket:open` — открытие корзины.
- `basket:remove` — удаление товара из корзины (передается `id` товара).
- `order:open` — открытие формы заказа.
- `order:next` — переход к следующему шагу оформления заказа.
- `order:submit` — отправка заказа.
- `buyer:changed` — вызывается при изменении данных покупателя.
